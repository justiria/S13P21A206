# SSAFY 과제

## ✅ Git Branch 개념

- `브랜치(branch)`는 독립적인 작업 공간을 만들어주는 Git의 기능입니다.
- *기존 코드(main)**를 그대로 두고 **새로운 기능/수정 작업**을 별도로 진행할 수 있도록 합니다.
- 각각의 브랜치는 **스냅샷(커밋)**들의 연속입니다.

📌 **비유**:

`main`은 "출시된 앱",

`branch`는 "새 기능을 실험 중인 복사본"입니다.

---

## ✅ 장점과 단점

### ✅ 장점

| 장점 | 설명 |
| --- | --- |
| 독립성 보장 | 기존 코드(main/master)에 영향 없이 작업 가능 |
| 협업 효율 | 개발자별 브랜치 분리로 충돌 최소화 |
| 실험/테스트 | 새 기능을 테스트했다가 버릴 수도 있음 |
| 코드 리뷰 용이 | Pull Request로 비교 가능 |
| 병합 관리 | 검토 후 병합(Merge) 가능, 안정성 확보 |

### ❌ 단점

| 단점 | 설명 |
| --- | --- |
| 병합 충돌 가능 | 동일 파일 수정 시 충돌 발생 가능 |
| 브랜치 관리 필요 | 너무 많은 브랜치가 생기면 관리 복잡 |
| 학습 곡선 | Git 사용이 익숙하지 않으면 실수하기 쉬움 |

## 🛠️ 기본 명령어 정리

### 🔧 브랜치 생성 및 이동

```bash
bash
복사편집
git branch 브랜치명          # 브랜치 생성
git checkout 브랜치명        # 브랜치로 이동
git switch 브랜치명          # (추천) 브랜치 이동
git switch -c 브랜치명       # (추천) 브랜치 생성 + 이동

```

### 🔧 브랜치 목록 확인

```bash
bash
복사편집
git branch                  # 로컬 브랜치 목록
git branch -r              # 원격 브랜치 목록
git branch -a              # 전체 브랜치 목록

```

### 🔧 브랜치 삭제

```bash
bash
복사편집
git branch -d 브랜치명      # 병합된 브랜치 삭제
git branch -D 브랜치명      # 강제 삭제

```

### 🔧 브랜치 병합

```bash
bash
복사편집
git merge 브랜치명          # 현재 브랜치에 다른 브랜치 병합

```

## 🧑‍💻 실전 예시

```bash
bash
복사편집
git switch -c feature/login     # 로그인 기능 브랜치 생성 및 이동
# 로그인 기능 개발 후 커밋 완료

git switch main                 # 메인 브랜치로 이동
git merge feature/login         # 로그인 기능 병합
git branch -d feature/login     # 병합 완료 후 브랜치 삭제

```

## 🚦 브랜치 활용 전략 예시

### 1. Git Flow

- `main`: 배포 버전
- `develop`: 개발 버전
- `feature/*`: 기능 개발
- `hotfix/*`: 긴급 수정
- `release/*`: 릴리스 준비

### 2. GitHub Flow

- `main` 기준으로 브랜치 생성 → 기능 개발 → PR 요청 → 코드 리뷰 후 병합
