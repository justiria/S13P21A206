# **Jira 사용법 정리**

## **Jira란?**

Jira는 Atlassian에서 제공하는 **프로젝트·이슈 관리 도구**로, 팀 내의 업무를 체계적으로 추적·관리하는 데 사용됩니다. 소프트웨어 개발 뿐만 아니라 다양한 팀 협업 환경에서 활용할 수 있습니다.

## **Jira 기본 용어**

- **이슈(Issue):** 하나의 작업, 과제, 버그 등 처리해야 할 항목
- **에픽(Epic):** 여러 개의 이슈(주로 스토리 또는 작업)를 포함하는 큰 작업 단위
- **스토리(Story):** 사용자의 요구사항이나 기능 개선 등의 업무 항목
- **태스크(Task):** 작은 단위의 작업
- **버그(Bug):** 문제점이나 수정이 필요한 사항
- **보드(Board):** 칸반(Kanban)·스크럼(Scrum) 방식 등 이슈의 상태와 흐름을 시각적으로 보여주는 곳
- **스프린트(Sprint):** 애자일(Agile) 개발에서 일정 기간 동안 진행되는 작업의 묶음
- **워크플로우(Workflow):** 이슈가 해결되기까지 거치는 상태(예: 할 일 → 진행 중 → 완료)
- **JQL (Jira Query Language):** 이슈 검색을 위한 고급 쿼리 언어

## **Jira 기본 사용법**

## **1. 계정 생성 및 프로젝트 만들기**

- 공식 사이트에서 Jira 계정 생성
- 대시보드에서 “프로젝트 생성” 클릭 후 템플릿(칸반, 스크럼 등) 선택
- 프로젝트 이름, 키, 팀원 초대 진행

## **2. 이슈 생성과 관리**

- 프로젝트 내에서 “이슈 생성” 클릭
- 이슈 유형(Task, Story, Bug 등), 제목, 설명, 우선순위, 담당자 입력
- 이슈의 상태(예: 할 일, 진행 중, 완료) 변경하며 진행 상황 관리

## **3. 애자일 보드 활용**

- 스크럼: 스프린트 단위로 반복적으로 작업 관리
- 칸반: 작업의 흐름을 보드 상에서 지속적으로 관리
- 실시간으로 이슈 드래그&드롭, 상태 변경, 팀원과 협업

## **4. 검색과 필터, 대시보드**

- 강력한 검색(JQL)과 필터 기능 사용해 이슈 관리
- 대시보드를 통해 프로젝트 현황 및 주요 지표 확인

## **Jira 활용 팁**

- **자동화**: 반복 작업을 룰(규칙)로 자동화하여 효율성 강화
- **리포트**: 진행률, 해결량, 시간 경과 등 다양한 리포트 제공
- **통합**: Slack, Git, Confluence 등 다양한 툴과 연동 가능
- **권한 설정**: 팀원별 역할/권한을 세밀하게 지정할 수 있음

## **참고: Jira 기본 사용 절차 요약**

| **단계** | **설명** |
| --- | --- |
| 계정 생성 | Jira 공식 사이트에서 계정 회원 가입 |
| 프로젝트 생성 | 템플릿 선택 후 프로젝트 정보 입력 |

| 리포트/대시보드 | 프로젝트 현황 및 주요 지표 시각화 |
| 검색/필터 | JQL, 필터로 원하는 이슈만 집계·조회 |
