# - WebRTC 정리

## WebRTC란?

> WebRTC (Web Real-Time Communication)
> 
> 
> 웹 브라우저 간에 **실시간 영상, 음성, 데이터** 통신을 가능하게 해주는 **오픈소스 기반 통신 기술**
> 
> 플러그인 없이 **P2P 방식으로 직접 통신** 가능
> 
- 주요 API:
    - `getUserMedia()` – 카메라/마이크 접근
    - `RTCPeerConnection` – 피어 간 연결, 스트리밍
    - `RTCDataChannel` – 텍스트/파일 등 데이터 전송

---

## WebRTC 구조 및 동작 순서

1. **미디어 캡처**
    
    `getUserMedia()`로 카메라/마이크 스트림 확보
    
2. **시그널링(Signaling)**
    
    P2P 연결을 위한 SDP, ICE 정보를 교환
    
    > WebSocket, HTTP 등 외부 채널로 구현
    > 
3. **RTCPeerConnection 생성**
    
    브라우저 간 직접 연결, STUN/TURN 서버 활용
    
4. **통신 시작**
    
    오디오/비디오/데이터 스트림 전송
    

---

## WebRTC 장점

| 장점 | 설명 |
| --- | --- |
| 실시간성 | 짧은 지연 시간 (Low Latency) |
| 플러그인 불필요 | 설치 없이 브라우저로 바로 사용 |
| 오픈소스 | 무료, 확장성 높음 |
| 다양한 데이터 지원 | 오디오, 비디오, 텍스트, 파일 전송 가능 |
| 보안 내장 | DTLS, SRTP 등 기본 암호화 포함 |

---

## WebRTC 단점

| 단점 | 설명 |
| --- | --- |
| 시그널링 직접 구현 | 연결 전 단계는 개발자가 처리해야 함 |
| NAT/방화벽 문제 | TURN 서버 필요 시 구축 부담 |
| 다자간 연결 어려움 | SFU/MCU 서버 별도 필요 |
| 브라우저 호환성 | 일부 환경에서 기능 제한 가능성 있음 |

## 주요 활용 사례

| 분야 | 활용 예시 |
| --- | --- |
|  영상 통화 | Zoom, Google Meet, Discord 등 |
| 온라인 교육 | 화상 강의, 실시간 피드백 |
| 고객지원 | 실시간 상담, 원격 화면 공유 |
| 게임 | 멀티플레이 음성 채팅 |
| 파일 공유 | P2P 전송, 예: Snapdrop |
| 원격 진료 | 실시간 영상 진단, 기기 제어 |

---

## 개인 소감 및 핵심 정리

- **브라우저만으로 실시간 통신 가능**하다는 점이 인상 깊었음
- 단순한 영상 통화 외에 **데이터 전송, 파일 공유 등 다양한 응용** 가능
- **시그널링과 방화벽 문제**는 직접 해결해야 해서 난이도 높음
- 하지만 **SFU 서버 조합(Mediasoup, Janus 등)** 으로 확장 가능성 큼
