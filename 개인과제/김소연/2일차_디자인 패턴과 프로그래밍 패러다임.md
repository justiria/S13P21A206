🎯 전략 패턴 (Strategy Pattern)
객체의 행위를 직접 수정하지 않고, 캡슐화된 알고리즘(전략)을 교체하여 유연하게 변경 가능

컨텍스트: 전략을 담고 있는 환경

✅ 사용 예시
결제 수단 변경: 네이버페이, 카카오페이 등

Node.js의 passport 인증 미들웨어

👀 옵저버 패턴 (Observer Pattern)
주체의 상태 변화를 옵저버들이 관찰하며, 변경 시 자동으로 알림 전파

✅ 사용 예시
트위터 알림 시스템

Vue 3.x의 ref, reactive → DOM 자동 갱신

이벤트 기반 시스템, MVC 패턴에서 유용

🧱 프록시 패턴 (Proxy Pattern)
대상 객체 접근을 가로채거나 필터링하는 중간 계층 제공

✅ 사용 예시
보안 검사, 로깅, 캐싱

프록시 서버: Cloudflare

CORS 우회 처리

🔁 이터레이터 패턴 (Iterator Pattern)
iterator를 통해 컬렉션 요소를 순차적으로 접근

✅ 장점
자료 구조와 무관하게 동일한 방식으로 순회 가능

🔓 노출 모듈 패턴 (Revealing Module Pattern)
즉시 실행 함수 (IIFE) 로 private / public 접근을 구분하여 정보 은닉 구현

✅ 사용 예시
JavaScript의 CommonJS (CJS) 모듈

🧩 아키텍처 패턴: MVC / MVP / MVVM
📌 MVC (Model - View - Controller)
Model: 데이터, 비즈니스 로직

View: 사용자 UI

Controller: 흐름 제어

✅ 예시: React, Java Spring

📌 MVP (Model - View - Presenter)
View ↔ Presenter 간 1:1 강한 결합

MVC보다 더 직관적인 연결 구조

📌 MVVM (Model - View - ViewModel)
ViewModel이 View를 추상화하여 분리

특징

커맨드 패턴 + 양방향 데이터 바인딩

UI 테스트 용이 / 재사용성 ↑

✅ 예시: Vue.js

💡 프로그래밍 패러다임
🧭 패러다임 분류
분류	설명
🗣️ 선언형 (Declarative)	무엇을 할 것인가 기술
🧾 명령형 (Imperative)	어떻게 할 것인가 순차적으로 명령
🔧 절차형 (Procedural)	명령형 하위, 함수 중심 흐름
🧱 객체지향형 (OOP)	객체 간 상호작용
🧠 함수형 (Functional)	수학적 함수 기반, 부작용 없음

☕ Java의 함수형 도입 (JDK 1.8~)
✅ 람다식, 메서드 레퍼런스 → 함수형 프로그래밍 가능

✅ Stream API → 선언형 스타일 가능

📌 함수형 프로그래밍 핵심
🔁 순수 함수: 입력이 같으면 출력도 같음, 부작용 없음

🎁 고차 함수: 함수를 인자로 받거나 반환함

🧩 일급 객체 (First-Class Function):

변수 할당 가능

인자로 전달 가능

반환값으로 사용 가능

🧱 객체지향 프로그래밍 (OOP)
객체 = 데이터(속성) + 기능(메서드)

🎯 핵심 개념
개념	설명
추상화	핵심 기능만 노출
캡슐화	데이터 + 메서드 결합, 정보 은닉
상속성	기능 재사용 및 확장
다형성	하나의 메서드/클래스가 다양한 방식으로 동작

🔍 다형성 종류
오버로딩: 같은 이름, 인자 다름 (정적 바인딩)

오버라이딩: 상속 메서드 재정의 (동적 바인딩)

🧱 SOLID 객체지향 설계 원칙
원칙	약어	설명
단일 책임 원칙	SRP	클래스는 하나의 책임만 가져야 함
개방-폐쇄 원칙	OCP	확장에는 열려 있고, 수정에는 닫혀 있어야 함

인터페이스 분리 원칙	ISP	인터페이스는 작고 구체적으로 분리
의존 역전 원칙	DIP	추상화에 의존, 세부 구현에 의존 X
