# 네트워크 기초 정리

> 서버·라우터·스위치 등의 **노드(Node)** 와  
> 유선·무선 **링크(Link)** 로 연결되어 리소스를 공유하는 집합

---


## 1. 기본 개념

- **처리량(Throughput)**  
  네트워크 내에서 성공적으로 전달된 데이터 양 (단위: bps)

- **트래픽(Traffic)**  
  특정 시점에 링크 내를 흐르는 데이터 양

- **대역폭(Bandwidth)**  
  주어진 시간 동안 흐를 수 있는 최대 비트 수

- **지연시간(Latency)**  
  메시지가 두 장치 사이를 왕복하는 데 걸린 시간

- **토폴로지(Topology)**
  - **링형(Ring)**  
    - 구조적 단순하지만 회선 장애 시 전체 영향  
  - **메시(Mesh)**  
    - 다중 경로 제공 → 장애 허용, 트래픽 분산

## 3. 성능 분석 명령어

| 명령어   | 역할                                                         |
| -------- | ------------------------------------------------------------ |
| `ping`   | ICMP 이용, 대상 노드에 패킷 전송 후 응답 시간 확인           |
| `netstat`| 현재 접속 서비스, 라우팅 테이블, 포트 등 네트워크 상태 확인  |
| `nslookup` | 도메인 ↔ IP 매핑 확인, DNS 오류 진단                     |
| `tracert` / `traceroute` | 목적지까지의 경유 경로(hops) 확인            |

> **병목 원인**: 대역폭, 토폴로지, 서버 CPU/메모리, 비효율적 구성

## 5. TCP/IP와 OSI 모델

| 계층 모델    | TCP/IP 4계층  | OSI 7계층        |
| ------------ | ------------- | ---------------- |
| 애플리케이션 | Application   | Application, Presentation, Session |
| 전송         | Transport     | Transport        |
| 인터넷       | Internet      | Network          |
| 링크         | Link          | Data Link, Physical |

- **TCP**: 연결지향·신뢰성 보장 (3-way 핸드셰이크, 순서 보장)
- **UDP**: 비연결·비신뢰성, 빠른 전송 (데이터그램)

### TCP 연결 과정

1. **3-way 핸드셰이크**  
   `SYN → SYN-ACK → ACK`  
2. **4-way 해제**  
   `FIN_WAIT_1 → CLOSE_WAIT → FIN_WAIT_2 → TIME_WAIT → CLOSED`

## 8. 네트워크 기기

| 계층 | 장치            | 역할                                  |
| ---- | --------------- | ------------------------------------- |
| L2   | 스위치(Bridge)  | MAC 주소 기반 프레임 포워딩           |
| L3   | 라우터(Router)  | IP 주소 기반 패킷 경로 설정           |
| L7   | 로드밸런서      | HTTP 헤더·쿠키 등으로 요청 분산·헬스체크 |
| L1   | 리피터, 허브    | 신호 증폭·복제                        |

## 9. IP 주소와 ARP

- **ARP**: IP → MAC 매핑  
- **hop-by-hop** 전송  
- **게이트웨이** 확인: `netstat -r`

---

## 10. HTTP 진화 과정

| 버전   | 특징                                                         |
| ------ | ------------------------------------------------------------ |
| **1.0**| - 연결당 1요청 → 높은 RTT<br> - 이미지 스프라이트, 압축 사용 |
| **1.1**| - Keep-Alive 지원 → 다중 요청<br> - HOL Blocking 발생, 헤더 무거움 |
| **2**  | - 멀티플렉싱(병렬 스트림)<br> - 헤더 압축(HPACK)<br> - 서버 푸시 |
| **3**  | - QUIC 기반(UDP)<br> - 0-RTT 연결<br> - TLS 1.3 내장          |
